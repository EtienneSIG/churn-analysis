## Tips for Testing

1. **Ask the same question in different ways.** A good Data Agent should handle synonyms:
   - "Churn rate" = "percentage of customers who left" = "% churned"
   - "Activity tier" = "transaction frequency group" = "how often they transact"

2. **Inspect the generated SQL.** After each answer, click "View query" to see the SQL the agent generated. This is the best way to validate correctness.

3. **Try follow-up questions.** After asking "Which activity tier has the highest churn rate?", ask "Why do you think that is?" — a good agent will explain based on the data.

4. **Test edge cases:**
   - "Show me customers with no products and no transactions" (should return near-zero if data is generated correctly)
   - "What is the churn rate for customers with more than 5 products?" (may be very few customers)

5. **Compare to Power BI.** Your Power BI report and the Data Agent should agree on key numbers. Any discrepancies indicate either an agent instruction issue or a data modeling issue.
